# noonoji-viewer 디자인 가이드 v1

## 목표
- 연재형 블로그를 빠르게 읽히게 만든다.
- 콘텐츠는 담백하게, UI는 일관되게 유지한다.

## 레퍼런스 믹스
- 레이아웃: Astro Paper 계열(읽기 중심)
- 컴포넌트: shadcn/Tailwind UI 계열(카드/콜아웃)
- 문장 리듬: 뉴스레터형(짧은 문단, 핵심 먼저)

---

## 1) 디자인 토큰

### 컬러
- 배경: `#F8FAFC`
- 본문: `#111827`
- 보조텍스트: `#6B7280`
- 보더: `#E5E7EB`
- 포인트: `#2563EB`
- 성공카드: `#F0FDF4` / 보더 `#BBF7D0`
- 경고카드: `#FFFBEB` / 보더 `#FDE68A`

### 간격/반경
- 간격 스케일: `8 / 12 / 16 / 24 / 32`
- 카드 라운드: `12px`
- 큰 카드 라운드: `16px`

### 타이포
- 제목 H1: 34px / 700
- 섹션 H2: 24px / 700
- 카드 제목 H3: 18px / 600
- 본문: 17px / line-height 1.75
- 문단 길이: 2~3문장

---

## 2) 페이지 구조

### 홈(`/`)
1. 히어로(한 줄 설명)
2. 카테고리 카드 3개(수업/업무일지/영상)
3. 최신 글 6개

### 목록(`/lessons`, `/worklog`, `/videos`)
- 카드 리스트
- 각 카드: 제목/날짜/요약/태그

### 상세(`/posts/[slug]`)
- 제목 + 날짜 + 태그
- 본문
- 중간 컴포넌트 2개 이상 권장(TIP, FlowCard)
- 하단: 다음 글/이전 글 링크

---

## 3) 컴포넌트 세트 (v1)

1. `PostCard`
- 목록 공통 카드

2. `InfoBox`
- variant: `tip | note | warning`

3. `MacWindow`
- 코드/용어 설명 블록

4. `FlowCard`
- 문제→원인→조치→결과

5. `CategoryBadge`
- lessons/worklog/videos 라벨

---

## 4) 콘텐츠 작성 규칙
- 첫 4줄에 핵심 결론 포함
- 전문용어는 본문이 아니라 콜아웃 박스로 분리
- 한 문단 2~3문장
- 문장 길이는 가능한 짧게

### 금지
- 긴 문단(5문장 이상)
- 장식용 이모지 남발
- 같은 의미 반복

---

## 5) 연재 운영 룰
- 주 2~3편 발행
- 글당 태그 3~5개
- 발행 전 체크:
  - [ ] 제목이 한 줄로 의미 전달되는가
  - [ ] 첫 문단에서 핵심이 보이는가
  - [ ] 카드 컴포넌트 2개 이상 들어갔는가
  - [ ] 모바일에서 줄바꿈이 깨지지 않는가
